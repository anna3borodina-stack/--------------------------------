# Vibe Coder — дизайн фронтенда (адаптация frontend-design)

Скилл сочетает идеи **distinctive frontend** (смелая концепция, детали, не generic) с **правилами проекта** ниже. При конфликте «красиво вообще» и «правила репозитория» — **сначала правила репозитория**, затем усиление в рамках них.

## Контекст продукта

- **Назначение**: одностраничное портфолио вайбкодера — сайты, интерфейсы, digital и AI-first проекты с характером.
- **Аудитория**: эксперты, креаторы, digital- и AI-продукты, те, кто не хочет безликий шаблон.
- **Тон по умолчанию**: тёмная тема, «craft + digital»: глубина фона, акценты (лайм / бирюза / тёплый коралл), моно для техничных деталей, ощущение собранности и вкуса, не кричащий маркетинг.

## Жёсткие ограничения проекта (не нарушать)

1. **Hero-заголовок** — шрифт **Manrope** (`--font-sans`), кегль `clamp`, жирность 700, градиент **только** на `.hero__accent`. Не менять семейство без явного запроса пользователя.
2. **Заголовки секций (h2)** — единый стиль `.section__title` и переменные `--heading-section-*`. Подзаголовки в карточках — `.skill-card__title`, `.steps__title`, `.project-card__title` в одной шкале.
3. **Стек**: статическая вёрстка **HTML + CSS + ванильный JS**; новые эффекты — в первую очередь CSS, при необходимости небольшой JS. Не вводить React/Vite «ради скилла», если пользователь не просил.
4. **Отзывы**: карусель с **одной видимой карточкой**; вертикальный сдвиг `.testimonials__track` в `.testimonials__viewport`, без наслоения карточек.
5. **Контент и порядок секций** — по правилу проекта `vibecoder-portfolio` / ТЗ; новые блоки только по явному запросу.
6. **Доступность**: учитывать `prefers-reduced-motion` — анимации не должны делать контент нечитаемым (hero, reveal, карусель).
7. **Файлы**: `index.html`, `css/style.css`, `js/main.js`; фото `images/hero.jpg` с fallback при ошибке загрузки.

## Как применять «дизайн-мышление» здесь

Перед правками зафиксировать направление (одно предложение):

- **Задача интерфейса** — усилить доверие и «вайб», не перегрузить.
- **Тон** в рамках бренда: например *refined dark*, *editorial tech*, *soft brutalism* — но палитра и шрифты hero остаются в рамках ограничений выше.
- **Запоминающийся штрих**: один сильный акцент (фон, крупная типографика вне hero, паттерн, микродвижение блока) лучше, чем десять слабых.

## Эстетика в рамках проекта

- **Типографика**: не подменять Manrope в hero. Для остального текста можно тонко настраивать иерархию (межстрочные интервалы, трекинг, контраст muted/основной), **JetBrains Mono** — для тегов, цифр шагов, чипов. Не уводить в Arial/Inter как «дефолт» — пара уже выбрана осознанно.
- **Цвет и тема**: опираться на CSS-переменные `--bg`, `--accent`, `--accent-teal`, `--accent-warm` и т.д.; усиливать доминанту и 1–2 акцента, не распылять равномерную «радугу».
- **Движение**: приоритет CSS (`transition`, `@keyframes`, `animation-delay` для stagger). В JS — только то, что нужно (карусель, курсор, reveal). Высокий эффект: один продуманный вход (hero уже частично это делает) + scroll-reveal; не размазывать десятки независимых микроанимаций.
- **Композиция**: асимметрия сетки hero, воздух между секциями, опционально диагональ/смещение декора **вне** критичных зон доступности. Не ломать читаемость длинных текстов.
- **Фон и детали**: шум, мягкое свечение курсора, градиентные пятна — в духе уже заложенного; усиливать атмосферу **слоями** (текстура, свет, бордеры), а не только сплошным цветом.

## Чего избегать (и здесь, и в generic frontend-design)

- Шаблонный «AI-стиль»: фиолетовые градиенты на белом, Inter как «универсальный» выбор, однотипные карточки без связи с контекстом.
- Конфликт с ТЗ: подмена смыслов секций, смена шрифта hero «для красоты», ломание карусели ради эффекта.

## Проверка перед сдачей

- Hero и h2 визуально согласованы; семейство hero — Manrope.
- Карусель: одна карточка в окне, без наложения.
- `prefers-reduced-motion`: контент не пропадает (особенно hero-строки).
- Изменения сфокусированы на запросе; лишние файлы и несвязанные рефакторинги не добавлять.

## Связанные правила проекта

- Контент и структура секций: `.cursor/rules/vibecoder-portfolio.mdc` (если есть в репозитории).
- Базовый скилл портфолио: логика файлов и технические заметки из цепочки курса по этому лендингу.
